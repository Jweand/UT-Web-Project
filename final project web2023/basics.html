In this lesson, we will give you a general idea of ​​some basic terms in the Java language,
noting that you will learn about each term in detail in subsequent lessons.
The basic terms in Java language are the following.


class

object

Instance variable

methods






Class concept

A class is a large container that can contain all the code, including variables, functions, and objects






Object concept

An object is an exact copy of a specific class.
Since an object is an instance of a class, we can say that an object cannot be created if there is no class.
So, in the concept of object programming, we create a specific class called blue print, meaning (the raw copy or the original copy),
and then we create one or more copies of this class and do what we want with it without changing the contents of the basic class,
and thus we have preserved the codes of the basic class because We modify the copy, not directly.

The basic idea in object programming is that you divide a single program into several classes,
and each class is programmed to implement a specific aspect of the program.
After completing all the program sections in several classes, we just create copies of them in the place we need them.
Thus, we collected all the classes that were written separately and linked them together as if we had written them in only one class.


Note:

The object is a copy of the class, not a copy of the file containing the class.
Also, the basic class is created without giving values to its variables







Instance variables concept

Every object we create from a class has its own copy of the variables in that class.
The variables of which a copy is given to each class object are what are called properties.
Example: If we have a class called do0o that contains two variables x and y,
and we create two objects from it, the first object is called A and the second is called B.

Here each object will contain a special copy of the objects in the do0o class.
Object A will have its own copy of the variables x and y that object B has nothing to do with.
Object B will have its own copy of the variables x and y that object A has nothing to do with.







Method in Java

general, a method is a way to perform some task.
Similarly, the method in Java is a collection of instructions that performs a specific task.
It provides the reusability of code.
We can also easily modify code using methods.





What is a method in Java?

A method is a block of code or collection of statements
or a set of code grouped together to perform a certain task or operation.

- It is used to achieve the reusability of code.
- We write a method once and use it many times.
- We do not require to write code again and again.
- It also provides the easy modification and readability of code,
just by adding or removing a chunk of code.

- The method is executed only when we call or invoke it.

The most important method in Java is the main() method
